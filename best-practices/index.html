# 最佳实践

程序的编写应遵循以下最佳实践经验，使程序保持更好的可维护性，持续健壮。

就近维护

一个页面或组件相关的内容应放置到最短引用路径范围内。

最小单元

一个页面、组件或函数过长，请考虑是否可以将其进一步拆解和封装为更小的单元。

单一目的

一个组件、类、函数应该只处理单一的事务，如果其中包含了分支逻辑，则考虑是否需要将其进一步拆解成更小的功能单元。

统一风格

当项目代码流转于不同开发者之间时，一致的代码风格能让所有人欢欣鼓舞。请遵循框架内置的 Eslint 规格配置。
